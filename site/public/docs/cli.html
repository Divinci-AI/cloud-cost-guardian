CLI Reference

@kill-switch/cli — monitor cloud spending, kill runaway services from the terminal. Designed for both developers and AI agents.

Install

# From npm (coming soon)
npm install -g @kill-switch/cli

# From source
git clone https://github.com/AiExpanse/kill-switch.git
cd kill-switch/packages/cli
npm install && npm run build
npm link  # makes `kill-switch` available globally

Quick Start

# 1. Authenticate
kill-switch auth login --api-key ks_live_your_key_here

# 2. Apply cost protection
kill-switch shield cost-runaway

# 3. Run a monitoring check
kill-switch check

# 4. View results
kill-switch accounts list

Authentication

The CLI uses personal API keys (prefixed with ks_). Create one from the web dashboard at app.kill-switch.net, or via the API.

Auth Resolution Order

  1. KILL_SWITCH_API_KEY environment variable (best for CI/CD and AI agents)
  2. --api-key flag on any command
  3. ~/.kill-switch/config.json file (set by kill-switch auth login)

kill-switch auth login --api-key KEY

Authenticate and save the key to ~/.kill-switch/config.json (0600 permissions).

kill-switch auth status

Show current auth status, account name, and tier.

kill-switch auth logout

Clear stored credentials.

Shield (Quick Protect)

The fastest way to add protection. One command applies a preset rule.

# Apply cost runaway protection
kill-switch shield cost-runaway

# List all available shields
kill-switch shield --list

# Apply with JSON output (for automation)
kill-switch shield gpu-runaway --json

Cloud Accounts

kill-switch accounts list

List all connected cloud accounts with provider, name, and status.

kill-switch accounts get <id>

Show detailed account info including thresholds and last check results.

kill-switch accounts add <provider> --name "My Account"

Connect a cloud provider. Supported: cloudflare, gcp, aws.

# Connect Cloudflare
kill-switch accounts add cloudflare \
  --name "Production" \
  --token "your-cf-api-token" \
  --account-id "your-account-id"

# Connect GCP
kill-switch accounts add gcp \
  --name "GCP Production" \
  --project-id "my-project" \
  --service-account '{"type":"service_account",...}'

kill-switch accounts delete <id>

Disconnect and delete a cloud account and its stored credentials.

kill-switch accounts check <id>

Run a manual monitoring check on a specific account.

Rules

kill-switch rules list

List all active kill switch rules.

kill-switch rules presets

List available preset rule templates.

kill-switch rules create <name> --trigger <type>

Create a custom rule. Trigger types: cost, security, api.

kill-switch rules toggle <id>

Enable or disable a rule.

kill-switch rules delete <id>

Delete a rule permanently.

Database Kill Switch

Multi-step database kill sequences: snapshot → verify → isolate → nuke. Each step is an explicit command — no interactive prompts.

# Initiate a kill sequence
kill-switch kill init \
  --credential-id "cred_abc123" \
  --trigger "credential compromise detected"

# Check status
kill-switch kill status dbkill-xyz789

# Advance to next step
kill-switch kill advance dbkill-xyz789 --credential-id "cred_abc123"

# Final step (nuke) requires human approval
kill-switch kill advance dbkill-xyz789 \
  --credential-id "cred_abc123" \
  --human-approval

# Abort if needed
kill-switch kill abort dbkill-xyz789

Alerts

kill-switch alerts list

List configured alert channels (PagerDuty, Discord, Slack, email, webhook).

kill-switch alerts test

Send a test alert to all configured channels.

Analytics

kill-switch analytics --days 30

FinOps analytics overview: daily costs, savings from kill-switch actions, violation trends.

Configuration

kill-switch config init

Create ~/.kill-switch/config.json with defaults.

kill-switch config list

Show all config values (API key is masked).

kill-switch config set <key> <value>

Set a config value. Keys: apiKey, apiUrl.

AI Agent Usage

The CLI is designed to be used by AI agents (Claude Code, etc.) without interactive prompts.

Agent-safe by design: Every parameter is passable as a flag. No interactive prompts unless explicitly requested. All output is parseable with --json. Exit codes are deterministic (0=success, 1=error, 2=auth error). 
# Set API key via env var (no config file needed)
export KILL_SWITCH_API_KEY=ks_live_abc123

# All commands work non-interactively with --json
kill-switch accounts list --json
kill-switch shield cost-runaway --json
kill-switch check --json

# Parse JSON output in scripts
kill-switch check --json | jq '.results[].violations | length'

Claude Code Integration

# In a Claude Code session:
KILL_SWITCH_API_KEY=ks_live_abc123 kill-switch shield cost-runaway --json
KILL_SWITCH_API_KEY=ks_live_abc123 kill-switch shield gpu-runaway --json
